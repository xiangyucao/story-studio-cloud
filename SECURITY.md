# Security policy

## Reporting a vulnerability

Please do not report security vulnerabilities in a public Issue or Discussion.

Use the repository's [private vulnerability reporting](https://github.com/xiangyucao/story-studio-cloud/security/advisories/new) so the maintainer can investigate before details become public.

Include the affected page or API, reproduction steps, potential impact, and any safe supporting material. Do not access, modify, or retain another user's private data while testing.

This project is maintained on a best-effort basis. Reports will be reviewed as time permits, and no response-time guarantee is offered.
